# mcp-server

> **Phase 9** — MCP Wire Protocol for External AI Agents

Exposes the **same `agent-toolset`** from Phase 8 to external AI agents (Claude Desktop, MCP Inspector, custom clients) via the [Model Context Protocol](https://modelcontextprotocol.io). Zero local tool definitions — the DRY proof that "one toolset, two surfaces" works.

## Transports

| Transport | Use Case | Configuration |
|-----------|----------|--------------|
| **HTTP + SSE** | Network demo (docker-compose, tunnel) | `MCP_TRANSPORT=http`, `MCP_STDIO_ENABLED=false` |
| **stdio** | Claude Desktop local subprocess | `docker run -i n11/mcp-server:dev` with env overrides |

The same Jib image serves both transports. Docker-compose runs HTTP; Claude Desktop overrides via `docker run -e`.

## Auth Bridge

External agents authenticate with `MCP_API_KEY`:

```
MCP_API_KEY --> identity-service /agents/exchange
                    |
                    v
              Internal RS256 JWT (24h, ROLE_USER)
                    |
                    v
              Gateway validates JWT normally
              X-User-Id propagated to all tool calls
```

- `AgentJwtClient` exchanges the API key for a JWT on first tool call
- `AgentJwtCache` caches the JWT with a 10-minute refresh buffer
- `JwtBearerInterceptor` attaches the JWT to all outbound HTTP calls

## Available Tools (10)

| Tool | Type | Description |
|------|------|-------------|
| `search_products` | read | Search product catalog |
| `get_product` | read | Get product details by ID |
| `list_categories` | read | List all categories |
| `add_to_cart` | mutate | Add product to cart |
| `view_cart` | read | View current cart |
| `update_cart_item` | mutate | Update cart item quantity |
| `remove_from_cart` | mutate | Remove item from cart |
| `create_order` | mutate | Create order from cart |
| `get_payment_link` | read | Get Iyzico checkout URL |
| `get_order_status` | read | Get order status |

All tools are imported from the `agent-toolset` shared module. **Zero** tool implementations exist in `mcp-server/src/main/java/`.

## DRY Proof

`McpServerToolsListEqualityTest` (in `infra-tests`) asserts that `mcp-server`'s `tools/list` response contains identical tool names and schemas as `ai-service` uses — CI breaks if someone duplicates a tool.

## Gateway Routing

MCP traffic enters via `/mcp/**` at the API gateway:
- `StripPrefix=0` — path forwarded as-is
- `response-timeout: -1` — long-lived connections
- JWT Bearer auth (exchanged via `/agents/exchange`, never raw API key)

## Required Environment Variables

| Variable | Description |
|----------|-------------|
| `MCP_API_KEY` | Agent API key (generated by AgentSeedRunner on first identity-service boot) |
| `MCP_TRANSPORT` | `http` (docker-compose) or `stdio` (Claude Desktop) |

### Getting the MCP_API_KEY

```bash
docker compose logs identity-service | grep "MCP_API_KEY="
# Copy the logged key into .env
```

## Build & Run

```bash
./gradlew :mcp-server:jibDockerBuild
docker compose up -d mcp-server
```

## Claude Desktop Configuration

```json
{
  "mcpServers": {
    "n11": {
      "command": "docker",
      "args": ["run", "-i", "--rm", "--network", "n11-net",
               "-e", "MCP_TRANSPORT=stdio",
               "-e", "MCP_STDIO_ENABLED=true",
               "-e", "MCP_API_KEY=<your-key>",
               "-e", "SPRING_CONFIG_IMPORT=configserver:http://config-server:8888",
               "-e", "EUREKA_CLIENT_SERVICEURL_DEFAULTZONE=http://eureka-server:8761/eureka/",
               "n11/mcp-server:dev"]
    }
  }
}
```
