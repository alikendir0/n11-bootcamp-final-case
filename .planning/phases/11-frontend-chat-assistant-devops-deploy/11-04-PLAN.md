| Boundary | Description |
|----------|-------------|
| GitHub Actions→GHCR | CI runner authenticates to package registry and publishes images. |
| GitHub Actions→Slack | CI runner sends build/release status to external webhook. |
| Repository YAML→secrets | Workflow files reference secrets but must not contain secret values. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-04-01 | I | Slack webhook | mitigate | Use only `${{ secrets.SLACK_WEBHOOK_URL }}`; grep rejects literal `hooks.slack.com`. |
| T-11-04-02 | E | GHCR publish | mitigate | Limit workflow permissions to `packages: write` and `contents: read`; use automatic `GITHUB_TOKEN`. |
| T-11-04-03 | T | release images | mitigate | Publish immutable `${{ github.ref_name }}` and `latest`; README in Plan 05 documents pinning immutable tags. |
| T-11-04-04 | R | CI status | mitigate | Slack message includes ref name and workflow run logs remain in GitHub Actions. |
</threat_model>
