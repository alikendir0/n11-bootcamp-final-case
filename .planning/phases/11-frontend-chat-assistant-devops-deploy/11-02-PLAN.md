## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SSE payload→DOM | Backend-generated summaries and product/card data are rendered in React. |
| chat card→cart API | Chat UI can trigger cart mutations through gateway-authenticated REST calls. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-02-01 | I | transcript rendering | mitigate | Render text as React text nodes only; no `dangerouslySetInnerHTML`. |
| T-11-02-02 | E | `ChatProductCard` | mitigate | Use existing token/auth path and login redirect; never accept userId from chat card props. |
| T-11-02-03 | T | card navigation | mitigate | Product routes are generated via existing `productUrlSegment`; payment external link only from backend `paymentPageUrl` data and rendered as explicit CTA. |
| T-11-02-04 | D | drawer streaming UI | mitigate | Disable send during active stream and keep retry scoped to last user message. |
</threat_model>
