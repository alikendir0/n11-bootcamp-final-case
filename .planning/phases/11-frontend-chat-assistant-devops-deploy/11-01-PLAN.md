<interfaces>
Backend stream contract:
```java
@PostMapping(produces = MediaType.TEXT_EVENT_STREAM_VALUE)
public SseEmitter stream(HttpServletRequest req, @Valid @RequestBody ChatRequest body)
public record ChatRequest(@NotNull UUID conversationId, @NotBlank String message) {}
```

Named SSE events emitted by ai-service: `delta`, `tool_call`, `tool_result`, `done`, `error`.
Payload keys:
- `delta`: `{ text, conversationId }`
- `tool_call`: `{ name, callId, argsJson }`
- `tool_result`: `{ callId, toolName, ok, summary, resultType, data }` where `resultType` is `products | product | cart | order | payment | generic` and `data` is a neutral JSON object/array derived from the executed tool result.
- `done`: `{ conversationId, finalText }`
- `error`: `{ code, messageTr }`

Gateway/auth pattern from `apiClient.ts`: `VITE_API_BASE_URL` + `/api/v1`, `Authorization: Bearer ${token}`, 401 clears token and dispatches `auth:unauthorized` with current redirect URL.
</interfaces>

## Trust Boundaries

| Boundary | Description |

| ai-service SSE→browser | Streaming JSON events from backend drive UI state and cart invalidation. |
| localStorage→React state | Stored conversation IDs/transcript data are untrusted browser-controlled data. |
