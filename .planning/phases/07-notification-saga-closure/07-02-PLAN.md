The InventoryRabbitConfig (full file at inventory-service/src/main/java/com/n11/inventory/messaging/InventoryRabbitConfig.java) is the canonical analog. Critical patterns:

```java
// Idempotent exchange re-declaration with service-prefixed bean name (Plan 05-04 disambiguation):
@Bean
public TopicExchange ordersExchangeForNotification() {
    return ExchangeBuilder.topicExchange("orders.tx").durable(true).build();
}

// Main queue + DLQ + binding triple per saga-contracts.md §3:
@Bean
public Queue notifyOrderConfirmedQueue() {
    return QueueBuilder.durable("notify.q.order-confirmed")
        .withArgument("x-dead-letter-exchange", "")
        .withArgument("x-dead-letter-routing-key", "notify.q.order-confirmed.dlq")
        .build();
}
@Bean public Queue notifyOrderConfirmedDlq() {
    return QueueBuilder.durable("notify.q.order-confirmed.dlq").build();
}
@Bean
public Binding notifyOrderConfirmedBinding(Queue notifyOrderConfirmedQueue,
                                           TopicExchange ordersExchangeForNotification) {
    return BindingBuilder.bind(notifyOrderConfirmedQueue)
        .to(ordersExchangeForNotification).with("order.confirmed");
}
```

Routing matrix (saga-contracts.md §7 event catalog + §2 queue table):

| Queue | Exchange (re-declared as bean) | Routing key | Producer service | Consumer |
|-------|--------------------------------|-------------|------------------|----------|
| notify.q.order-confirmed | orders.tx (ordersExchangeForNotification) | order.confirmed | order-service | notification-service |
| notify.q.order-cancelled | orders.tx (ordersExchangeForNotification) | order.cancelled | order-service | notification-service |
| notify.q.payment-failed | payments.tx (paymentsExchangeForNotification) | payment.failed | payment-service | notification-service |
| notify.q.user-registered | identity.tx (identityExchangeForNotification) | user.registered | identity-service | notification-service |

DLQ pattern (saga-contracts.md §3): every main queue uses `x-dead-letter-exchange=""` + `x-dead-letter-routing-key=<queue>.dlq`; DLQs are simple durable queues with no further routing (PATTERNS.md §DLQ Queue Declaration). Consumers running on `RabbitRetryConfig.sagaRetryInterceptor` from common-events get 3 attempts (1s/5s) before DLQ — saga-contracts.md §4 LOCKED policy.

Bean-name disambiguation (Plan 05-04 STATE.md decision): when infra-tests loads notification-service + cart-service + inventory-service on the same classpath (Plan 07-05), Spring rejects duplicate `@Bean` names. Inventory-service uses `ordersExchange()` and cart-service uses `ordersExchangeForCart()` — notification-service MUST use `ordersExchangeForNotification()`, `paymentsExchangeForNotification()`, `identityExchangeForNotification()` to avoid clashes.
