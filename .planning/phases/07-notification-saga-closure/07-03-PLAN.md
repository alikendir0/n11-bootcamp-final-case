Each listener:
1. Decodes UTF-8 body to Envelope; throws AmqpRejectAndDontRequeueException on JSON failure
2. Parses envelope.eventId() to UUID; throws AmqpRejectAndDontRequeueException on parse failure
3. Null-checks envelope.payload(); throws on null
4. Deserializes payload via objectMapper.treeToValue(envelope.payload(), <Payload>.class); throws on failure or null
5. Calls notificationService.handle*(eventId, envelope, payload) — propagates transient exceptions for StatefulRetryInterceptor 3x retries → DLQ

Service handler shape — modeled on cart-service/src/main/java/com/n11/cart/messaging/CartSagaService.java (lines 35–47), updated for ARCHITECTURE.md §2.10 schema (payload_json JSONB blob assembly):
```java
@Transactional
public void handleOrderConfirmed(UUID eventId, Envelope envelope, OrderConfirmedPayload payload) {
    if (processedEventRepository.existsById(eventId)) {
        LOG.debug("notification.service: duplicate event {}, skipping", eventId);
        return;
    }
    String subject = NotificationTemplates.SUBJECT_ORDER_CONFIRMED;       // "Siparişiniz onaylandı"
    String bodyTurkish = NotificationTemplates.bodyOrderConfirmed(payload);
    UUID correlationId = parseUuidOrNull(envelope.correlationId());
    if (correlationId == null) {
        // ARCHITECTURE.md §2.10 says correlation_id is NOT NULL; missing/invalid envelope correlationId
        // is a contract violation — fall back to eventId so the row still persists. The structured log
        // emits a WARN line for operator visibility (NOTIF-02 contract).
        correlationId = eventId;
        LOG.warn("notification.service: missing correlationId for event {} — using eventId fallback for audit row", eventId);
    }
    String payloadJson = renderPayloadJson(subject, bodyTurkish, envelope, payload);  // structured JSONB
    Notification n = new Notification(
        UUID.randomUUID(), payload.userId(), correlationId,
        "email-stub", "order.confirmed", payloadJson, "SENT", Instant.now());
    notificationRepository.save(n);
    processedEventRepository.save(new ProcessedEvent(eventId, "OrderConfirmedConsumer", envelope.eventType()));
    LOG.info("notification.sent recipient={} subject=\"{}\" correlationId={} eventType=order.confirmed channel=email-stub",
             payload.userId(), subject, envelope.correlationId());
}

// Helper: assemble a stable JSONB blob — keys: subject, bodyTurkish, eventEnvelope (the saga envelope as-is)
private String renderPayloadJson(String subject, String bodyTurkish, Envelope envelope, Object payload) {
    try {
        var node = objectMapper.createObjectNode();
        node.put("subject", subject);
        node.put("bodyTurkish", bodyTurkish);
        node.set("eventEnvelope", objectMapper.valueToTree(envelope));
        node.set("eventPayload", objectMapper.valueToTree(payload));
        return objectMapper.writeValueAsString(node);
    } catch (Exception e) {
        // Should not happen — Jackson on POJOs/records never throws here in practice
        throw new IllegalStateException("Failed to assemble payload_json for event " + envelope.eventId(), e);
    }
}
```

Notification entity (ARCHITECTURE.md §2.10) — JPA mapping for `notifications(id, user_id NULLABLE, correlation_id NOT NULL, channel, type, payload_json JSONB, status, sent_at)`:

```java
@Entity(name = "NotificationAudit")
@Table(name = "notifications")
public class Notification {
    @Id @Column(updatable = false, nullable = false)                        UUID id;
    @Column(name = "user_id")                                                UUID userId;            // NULLABLE
    @Column(name = "correlation_id", nullable = false)                       UUID correlationId;
    @Column(nullable = false)                                                String channel;
    @Column(nullable = false)                                                String type;            // event_type
    @Column(name = "payload_json", nullable = false, columnDefinition = "jsonb")
    @org.hibernate.annotations.JdbcTypeCode(org.hibernate.type.SqlTypes.JSON)
    String payloadJson;
    @Column(nullable = false)                                                String status;          // 'SENT' for v1
    @Column(name = "sent_at", nullable = false, updatable = false)           Instant sentAt;
    // protected no-arg ctor for JPA + 8-arg public ctor matching the field order above
}
```

Notes on JSONB mapping:
- Hibernate 6 (Spring Boot 3.5.14 BOM) supports `@JdbcTypeCode(SqlTypes.JSON)` natively for PostgreSQL JSONB. The Java field type can be `String` (raw JSON) for v1 — there's no need to deserialize into a typed POJO since the column is purely for audit/forensics, never read by application code.
- If `@JdbcTypeCode(SqlTypes.JSON)` triggers any classpath issue (e.g., Hibernate type resolution), fall back to a plain `@Column(name="payload_json", nullable=false, columnDefinition="jsonb") @org.hibernate.annotations.Type(JsonType.class) String payloadJson` after adding `io.hypersistence:hypersistence-utils-hibernate-63` — but the native annotation should work. Verify by running `./gradlew :notification-service:test --tests OrderConfirmedConsumerIdempotencyTest` (Plan 07-04 — Testcontainers Postgres).

                throw new AmqpRejectAndDontRequeueException("Invalid eventId: " + envelope.eventId(), e);
            }
            if (envelope.payload() == null || envelope.payload().isNull()) {
                LOG.error("notification.consumer.order-confirmed: null payload for event {}, routing to DLQ", eventId);
                throw new AmqpRejectAndDontRequeueException("Null payload for event " + eventId);
            }
            OrderConfirmedPayload payload;
            try {

            } catch (Exception e) {

            }
            if (payload == null) {
                LOG.error("notification.consumer.order-confirmed: null deserialized payload for event {}", eventId);

            }
