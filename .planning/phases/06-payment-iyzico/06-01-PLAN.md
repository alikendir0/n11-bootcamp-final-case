## Trust Boundaries
| Boundary | Description |
|----------|-------------|
| env/config → payment-service | Secrets and public callback base URL enter the service process |
| payment-service → Iyzico | Outbound sandbox API calls with merchant credentials |

## STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Information Disclosure | `payment-service.yml` | mitigate | Use `${IYZICO_API_KEY}`, `${IYZICO_SECRET_KEY}`, `${PUBLIC_BASE_URL}` placeholders only; no literal secrets. |
| T-06-02 | Tampering | `IyzicoProperties.callbackUrl()` | mitigate | Derive callback URL from one `PUBLIC_BASE_URL` per D-02; unit-test slash normalization. |
| T-06-03 | Repudiation | Payment persistence | mitigate | Store provider token/payment id and terminal timestamps for callback audit. |
</threat_model>
